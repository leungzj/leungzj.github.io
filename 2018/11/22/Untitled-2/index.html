关系型数据库基础理论

2018-11-22 数据库 mysql Comments

关系型数据模型有4种：
1、关系模型
2、实体-关系模型：将数据拆分为多个不同实体（数据表），通过实体之间建立联系，也叫E-R模型
3、对象关系模型：基于对象的数据模型，比如图片
4、半结构化数据模型：比如XML（扩展标记语言），存储数据的同时也存储数据的属性
而关系型数据库能够处理以上4种关系型数据模型

关系型数据库对象：
库、表、索引、视图、用户、存储过程、存储函数、触发器、事件调度器（类似cron）、约束

约束的分类：
1、键约束：数据类型约束
2、外键约束：引用完整性约束（使本表与另外的表关联起来的字段）
3、主键约束：此字段能唯一标识本表，并且不能为空，一张表只能有一个主键约束
4、唯一约束：此字段的值只能出现一次，可以为空，一张表可以有多个唯一约束
5、检查性约束：除了用域约束来规定数据的类型之外，检查性约束可以规定数据不违反常理（比如规定一个人的年龄不超过150岁）

关系型数据库分层：
表示层：表
逻辑层：存储引擎
物理层：数据文件

数据存储和查询组件：
1、查询管理器
DDL解释器
DML解释器
查询执行引擎
2、存储管理器
缓冲区管理器
文件管理器
事务管理器
权限和完整性管理器

SQL语句分类：
1、DML，数据操作语言（增删改查）
INSERT
DELETE
UPDATE
SELECT
2、DDL，数据定义语言（操作对象）
CREATE
DROP
ALTER
3、数据控制语言（访问权限）
GRANT
REVOKE

SQL语句执行过程（按步骤执行）：
1、连接管理器
2、线程管理器
3、用户模块：验证身份
4、命令分发模块：缓存模块（因为对数据操作都是需要把数据从磁盘读到内存，而内存往往比磁盘小得多，因此缓冲区管理器是提供了一种缓冲置换策略，将不常用的或者是已经过期很久的一些数据置换出去，以此来腾出内存空间）、日志模块（记录日志）
5、解释器
6、优化器（SELECT）、表定义模块（UPDATE、DELETE、INSERT）、表维护模块（检查、修改、备份、修复、优化，优化指的是碎片整理等）、复制模块、状态报告模块
7、访问控制模块：检查用户是否有权限操作相关数据表
8、表管理器：完成SQL操作，负责创建、读取或修改表结构定义文件，维护表描述符高速缓存，管理表锁
9、存储引擎：数据库跟磁盘上的数据打交道的接口

备注：
1、MySQL是单进程多线程的，多线程包括守护线程和应用线程
守护线程：MySQL内部的线程
应用线程：比如连进来的用户（每一个用户连进来都建立一个线程，一个线程不能同时为多个用户服务，因为这涉及到权限交叉的问题）
2、在32位系统中，MySQL只能使用到2.7G的内存（因此在生产环境中应该使用64位的系统）
3、MySQL跟Oracle的最大区别在于，MySQL支持插件式的存储引擎，也就是用户可以自行选择使用哪个存储引擎
4、5.5.8之前，MySQL的默认存储引擎是MYISAM，MYISAM不支持事务（MYISAM适用于查询比较多而修改比较少的场景，特别适用于数据仓库）；5.5.8之后，也就是MySQL被Oracle收购之后，MySQL的默认存储引擎变为InnodeDB（InnodeDB适用于在线事务管理系统，比如在线论坛等）
5、对于定长的数据来说，查询的效率高，但是很容易造成空间浪费；对于变长的数据来说，查询效率慢，但是可以节省存储空间。因此在定义变长还是定长的时候，应该选择折中的方案。
6、数据字典的定义：就是存储元数据的数据表。在MySQL初始化中，默认会生成一个叫做mysql的数据表，这个数据表就是用来存储数据库的相关元数据的，这个数据表也就是数据字典
